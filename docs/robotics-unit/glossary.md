# Glossary

This glossary defines the italicized terms from `README.md`.

- **dynamics**: The equations describing how robot or world states evolve over time under actions.
- **state**: A set of variables that capture the minimal necessary information describing a system at a given time.
- **control**: The commands or inputs applied to a robot to influence its state evolution.
- **actuators**: Physical components (motors, servos, etc.) that execute control signals on the robot.
- **optimal control problem**: A problem of finding control inputs that minimize a cost function subject to system dynamics.
- **actions**: Higher-level control choices or commands, often used in decision-making and reinforcement learning.
- **reward**: A scalar signal representing how desirable a state or outcome is, used in reinforcement learning.
- **return**: The accumulated reward over time that an agent aims to maximize.
- **reinforcement learning**: A learning paradigm where an agent learns policies from interaction with an environment via rewards.
- **perception**: The process of sensing and interpreting data about the robot's environment and state.
- **perception, planning control loop**: The iterative cycle of sensing, planning decisions, and executing actions.
- **policy**: A function or rule mapping states to control actions.
- **robust**: The property of a policy remaining effective under uncertainties or variation.
- **adaptive**: The property of a policy or system that can change based on new data or environmental changes.
- **agent**: An autonomous decision-making entity (e.g., robot) acting in an environment.
- **multi-agent**: Systems involving multiple interacting agents.
- **partial observability**: A situation where the full state of the environment is not directly available.
- **probabilistic beliefs**: Statistical representations of uncertainty over state estimates.
- **inferring**: Estimating hidden variables (like true state) from observed data.
- **sense**: The act of gathering sensor data about the environment.
- **plan**: The act of computing a sequence of actions to achieve a goal.
- **act**: The execution of control actions in the environment.
